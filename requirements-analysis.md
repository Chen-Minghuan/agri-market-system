# 2. 需求分析

## 2.1 需求概述

### 2.1.1 项目背景

本系统是一个面向乡村助农场景的农产品信息展示与售卖管理系统，旨在通过现代化的信息技术手段，帮助农户便捷地展示和销售农产品，同时为消费者提供透明、可信的农产品溯源信息和便捷的购买渠道。项目采用全栈开发模式，后端使用 Spring Boot 3 + Java 17，前端使用 Vue 3 + Element Plus，实现农产品从生产到销售的完整数字化流程。

### 2.1.2 建设目标

1. **提升农产品销售效率**：为农户提供便捷的农产品上架、管理和销售工具，降低销售门槛
2. **保障农产品溯源可信**：建立产地信息公示机制，增强消费者对农产品的信任度
3. **优化消费体验**：为消费者提供清晰的分类展示、便捷的下单流程和透明的订单追踪
4. **数据化经营决策**：为管理员和农户提供销量统计和数据分析功能，辅助经营决策

### 2.1.3 核心功能需求

#### 1. 用户管理与权限控制
- 支持三种角色：管理员、农户、消费者
- 实现基于角色的访问控制（RBAC）
- 提供用户注册、登录、登出功能
- 采用 Token 鉴权机制，保障系统安全性

#### 2. 农产品管理
- 农产品信息的增删改查（CRUD）功能
- 支持农产品分类管理
- 实现农产品上下架状态控制
- 支持图片上传功能，展示农产品封面
- 农产品信息包含：名称、价格、库存、单位、描述、分类、产地等

#### 3. 产地信息管理
- 维护产地/基地信息
- 公示农户联系方式
- 展示资质认证信息
- 提供产地介绍文本

#### 4. 订单管理
- 消费者自助下单功能
- 自动扣减库存、累加销量
- 订单状态流转管理（待付款→待发货→已发货→已完成/已取消）
- 取消订单时自动回滚库存和销量
- 支持订单查询和详情查看

#### 5. 数据统计与展示
- 经营总览（商品数、订单数、累计销量、销售额）
- 各品类销量与销售额统计
- 销量 TOP 商品排行
- 管理员查看全局数据，农户查看本人数据

### 2.1.4 非功能性需求

#### 1. 性能需求
- 系统响应时间：页面加载不超过 3 秒，接口响应不超过 500ms
- 支持并发用户数：至少 100 个同时在线用户
- 数据库查询优化，确保复杂统计查询性能

#### 2. 安全性需求
- 用户密码使用 BCrypt 加密存储
- 所有操作接口需 Token 鉴权
- 实现基于角色的权限控制
- 防止 SQL 注入、XSS 攻击等常见安全威胁

#### 3. 可用性需求
- 界面简洁直观，操作流程清晰
- 支持响应式布局，适配不同屏幕尺寸
- 提供友好的错误提示和操作反馈

#### 4. 可维护性需求
- 采用模块化设计，代码结构清晰
- 遵循 RESTful API 设计规范
- 提供完善的日志记录和异常处理机制
- 数据库脚本完整，便于系统部署和迁移

## 2.2 需求用例

### 2.2.1 管理员用例

#### UC-01: 用户管理
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入用户管理页面
  2. 查看所有注册用户列表
  3. 可以查看用户详情（用户名、角色、注册时间等）
  4. 可以修改用户信息
  5. 可以禁用或启用用户账户
- **后置条件**：用户信息更新完成
- **角色权限**：管理员

#### UC-02: 分类管理
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入分类管理页面
  2. 查看所有农产品分类列表
  3. 点击"新增分类"，填写分类名称和描述
  4. 点击"修改"，编辑已有分类信息
  5. 点击"删除"，删除指定分类（需确认）
  6. 点击"保存"完成分类管理
- **后置条件**：分类信息更新完成
- **角色权限**：管理员

#### UC-03: 产地管理
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入产地管理页面
  2. 查看所有产地信息列表
  3. 点击"新增产地"，填写产地名称、基地信息、农户信息、联系方式、资质认证、产地介绍
  4. 点击"修改"，编辑已有产地信息
  5. 点击"删除"，删除指定产地（需确认）
  6. 点击"保存"完成产地管理
- **后置条件**：产地信息更新完成
- **角色权限**：管理员

#### UC-04: 农产品管理
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入农产品管理页面
  2. 查看所有农产品列表（支持按分类、状态、关键词筛选）
  3. 点击"新增农产品"，填写农产品信息（名称、价格、库存、单位、分类、产地、封面图片、描述）
  4. 点击"修改"，编辑已有农产品信息
  5. 点击"上下架"，切换产品状态（上架/下架）
  6. 点击"删除"，删除指定农产品（需确认）
  7. 点击"保存"完成农产品管理
- **后置条件**：农产品信息更新完成
- **角色权限**：管理员

#### UC-05: 订单管理
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入订单管理页面
  2. 查看所有订单列表（支持按状态、时间筛选）
  3. 点击"订单详情"，查看订单完整信息（订单号、商品信息、收货信息、支付状态、发货状态）
  4. 点击"修改状态"，将订单状态从待付款改为待发货
  5. 点击"修改状态"，将订单状态从待发货改为已发货
  6. 点击"修改状态"，将订单状态从已发货改为已完成
  7. 点击"修改状态"，将订单状态改为已取消（库存自动回滚）
- **后置条件**：订单状态更新完成
- **角色权限**：管理员

#### UC-06: 销量统计
- **前置条件**：管理员已登录系统
- **基本流程**：
  1. 管理员进入销量统计页面
  2. 查看经营总览数据（商品总数、订单总数、累计销量、总销售额）
  3. 查看各品类销量与销售额柱状图
  4. 查看销量 TOP 商品排行榜
  5. 可按时间范围筛选统计数据
- **后置条件**：统计数据展示完成
- **角色权限**：管理员

  7. 点击"保存"完成农产品管理

#### UC-10: 我的销量统计

- **基本流程**：
  1. 农户进入我的销量统计页面
  2. 查看个人经营总览数据（商品总数、订单总数、累计销量、总销售额）
  3. 查看各品类销量与销售额柱状图（仅统计自己的产品）
  4. 查看销量 TOP 商品排行榜（仅统计自己的产品）
  5. 可按时间范围筛选统计数据
- **后置条件**：统计数据展示完成
- **角色权限**：农户

### 2.2.3 消费者用例

#### UC-11: 用户注册
- **前置条件**：消费者未登录系统
- **基本流程**：
  1. 消费者进入注册页面
  2. 填写用户名、密码、确认密码、手机号等信息
  3. 点击"注册"按钮
  4. 系统验证信息有效性
  5. 注册成功后自动登录，跳转到首页
- **后置条件**：消费者账户创建成功
- **角色权限**：消费者（自助注册）

#### UC-12: 用户登录
- **前置条件**：用户未登录系统
- **基本流程**：
  1. 用户进入登录页面
  2. 输入用户名和密码
  3. 点击"登录"按钮
  4. 系统验证用户信息
  5. 登录成功后，根据用户角色跳转到相应首页
  6. Token 保存到本地存储
- **后置条件**：用户登录成功
- **角色权限**：所有角色

#### UC-13: 商品浏览与搜索
- **前置条件**：消费者已登录系统
- **基本流程**：
  1. 消费者进入商品列表页面
  2. 浏览所有上架的农产品列表
  3. 通过分类筛选器选择分类
  4. 通过关键词搜索框输入关键词进行搜索
  5. 查看商品详细信息（名称、价格、库存、描述、产地信息）
- **后置条件**：商品浏览完成
- **角色权限**：消费者

#### UC-14: 商品详情查看
- **前置条件**：消费者已登录系统
- **基本流程**：
  1. 消费者点击某个商品进入详情页面
  2. 查看商品完整信息（名称、价格、库存、单位、分类、产地、封面图片、描述）
  3. 查看产地信息（基地、农户、联系方式、资质认证、产地介绍）
  4. 查看销量信息
  5. 点击"立即购买"按钮
- **后置条件**：商品详情查看完成
- **角色权限**：消费者

#### UC-15: 立即购买
- **前置条件**：消费者已登录系统，商品库存充足
- **基本流程**：
  1. 消费者在商品详情页面点击"立即购买"
  2. 确认购买数量和收货信息
  3. 点击"确认下单"
  4. 系统验证库存，创建订单
  5. 自动扣减库存，累加销量
  6. 订单创建成功，跳转到订单详情页面
- **后置条件**：订单创建成功，库存已扣减
- **角色权限**：消费者

#### UC-16: 我的订单查询
- **前置条件**：消费者已登录系统
- **基本流程**：
  1. 消费者进入我的订单页面
  2. 查看自己的所有订单列表
  3. 点击"订单详情"，查看订单完整信息
  4. 查看订单状态流转记录
  5. 可按订单状态筛选订单
- **后置条件**：订单信息查询完成
- **角色权限**：消费者

#### UC-17: 退出登录
- **前置条件**：用户已登录系统
- **基本流程**：
  1. 用户点击"退出登录"按钮
  2. 系统清除本地 Token
  3. 跳转到登录页面
- **后置条件**：用户已退出登录
- **角色权限**：所有角色
